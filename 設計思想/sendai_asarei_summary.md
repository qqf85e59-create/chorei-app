## 1. 設計の全体像

本朝礼は、仙台事務所メンバー8名の「関係性の質」を高めることを目的とした継続的な場である。ダニエル・キムの成功循環モデルに基づき、関係性の質の向上が将来的な業務上の健全な議論の土台となることを想定している。

運営は3フェーズに分けて段階的に進行する。

| フェーズ | 期間 | 目的 | 形式 |
|---------|------|------|------|
| 第一フェーズ | 5月〜6月（2か月） | 個人理解期 | 1名発話＋7名沈黙聴取 |
| 第二フェーズ | 7月〜9月（3か月） | 中間接続期 | 発話者＋応答者の2者構造 |
| 第三フェーズ | 10月〜 | 議論期 | 詳細は移行時に設計 |

## 2. 目的と設計思想

### 2.1 目的

仙台事務所メンバー間の「関係性の質」を高める。お互いをよく知り、率直に話せる関係をつくる。

### 2.2 設計思想の核

設計は以下の原則に基づく。

**原則一　場の性質を設計する**

参加者の振る舞いを細かく規定するのではなく、参加者が「判定されない」と体感できる場の構造を作ることで、率直さを引き出す。

**原則二　段階的な関係性の醸成**

個人理解から業務接続、議論への移行を段階化する。各段階で異なる形式を設定し、前段階の成果を次段階の基盤とする。

**原則三　評価と独立した場**

朝礼は業務評価とは独立した場として運営する。発話内容の記録、参加者のパーソナルデータ収集は行わない。

**原則四　運営の透明性**

運営の判断は常にグランドルールに基づき、問われれば根拠を説明する。運営の恣意性を排除する。

### 2.3 設計上の判断の経緯

当初案では「毎日・短時間・1対1の応答」を想定していたが、検討の結果、以下の転換を行った。

- 頻度を週3回に抑制（継続可能性の確保）
- 応答者の形式を段階化（第一フェーズは沈黙聴取、第二フェーズは「問いを置く」）
- 階層影響の緩和設計（発話順の反転、ペアリングの工夫）
- 発話内容の記録を行わない判断（設計思想との整合性）

## 3. 組織構成

仙台事務所の参加者は8名。等級構成は以下のとおり。

| 等級 | 人数 | 略称 |
|------|------|------|
| 等級5 | 1名 | E5 |
| 等級4+ | 1名 | E4p |
| 等級4 | 1名 | E4 |
| 等級3 | 3名 | E3a, E3b, E3c |
| 等級2 | 2名 | E2a, E2b |

### 3.1 等級構造からの含意

- 上位層3名（E5・E4p・E4）の存在感が場に与える影響は大きい
- 中位層3名（E3）は上位への忖度と下位への配慮の両方を抱える
- 下位層2名（E2）の「守られ感」の設計が第二フェーズ以降の鍵

### 3.2 等級5の姿勢

等級5は朝礼の発起人であり、フランクな姿勢を持つ。グランドルールへの遵守を明示的に表明する想定。設計の成否を左右する重要な前提条件。

## 4. グランドルール

### 4.1 本文

> 朝礼について
>
> この朝礼は、仙台事務所のメンバー同士の「関係性の質」を高める場です。お互いをよく知り、率直に話せる関係をつくっていきたいと考えています。参加にあたって、以下を共通のルールとします。
>
> 話す人は、テーマの範囲内で、何をどこまで話すかを自分で決めてかまいません。
>
> 聴く人は、話された内容について、その場で評価や感想を述べません。反応の仕方はフェーズごとに運営から案内する形式に沿ってください。
>
> 運営は、進行の判断について質問された場合、どのルールに基づく判断かを説明します。
>
> 出欠は、参加を原則とします。休むとき・途中で抜けるときは運営に連絡してください。理由は問いません。

### 4.2 条項の意図

| 条項 | 意図 |
|------|------|
| 前文 | 場の目的の明示。専門用語を補足する平易な文体を採用 |
| 第1条（話す人） | 発話者の自己決定原則。深度・具体性・抽象度は発話者が決める |
| 第2条（聴く人） | 聴取の無裁定原則。フェーズごとの形式指定で運用 |
| 第3条（運営） | 運営の透明性。判断根拠の説明義務 |
| 第4条（出欠） | 参加原則だが、欠席時の理由は問わない |

### 4.3 導入プロセス

- 本番開始前に全員に文面を事前配布
- 第1回朝礼で説明・確認
- 以降、起動時画面・メニューからいつでも参照可能

## 6. 第一フェーズ詳細設計

### 6.1 基本情報

- 期間：5月〜6月（2か月）
- 頻度：週3回（火・木・金）×8週＝24回
- 1回時間：10分
- 形式：通信会議
- 発話者：1名（輪番）
- 聴取者：7名
- 運営：ゆ氏（発話者と進行役は分離）
- 1人あたり発話回数：3回

### 6.2 1回の進行（10分）

| 区分 | 時間 | 内容 |
|------|------|------|
| 冒頭 | 30秒 | 運営が発話者・主題を告知 |
| 発話 | 5分 | 発話者が主題について話す |
| 関心表明 | 2分 | 聴取者7名が15〜20秒ずつ「関心を持った一点」を述べる |
| 締め | 1分30秒 | 運営が締めの挨拶、次回の告知 |

### 6.3 発話順序

**1巡目（5月第1週〜第3週、計8名＋予備1回）**

下位から上位の順で発話。最上位者の発話が場の基準になることを避け、反転効果を狙う。

| 回 | 曜日 | 発話者 |
|----|------|--------|
| 1 | 5月第1週火 | E2a |
| 2 | 5月第1週木 | E2b |
| 3 | 5月第1週金 | E3a |
| 4 | 5月第2週火 | E3b |
| 5 | 5月第2週木 | E3c |
| 6 | 5月第2週金 | E4 |
| 7 | 5月第3週火 | E4p |
| 8 | 5月第3週木 | E5 |
| 9 | 5月第3週金 | 予備（運営内棚卸し） |

**2巡目・3巡目**

運営側がランダムに決定、前日朝までに対象者に通知。予測可能性を下げて場の緊張感を保つ。

### 6.4 主題（週ごとに指定）

| 週 | 主題 |
|----|------|
| 第1週 | 最近気になっている景色や場所 |
| 第2週 | 休日に時間を使っている対象 |
| 第3週 | 最近ちょっといいなと思ったもの |
| 第4週 | 子どもの頃に夢中だったこと |
| 第5週 | 最近小さく困っていること |
| 第6週 | 自分がよく行く店 |
| 第7週 | 最近考えている問い（業務外） |
| 第8週 | 最後に読んだ本・観た作品・聴いた音楽 |

主題の選定基準：金額・規模・社会的評価が比較軸にならない。業務文脈と距離がある。判断基準・関心が自然に現れる。

### 6.5 聴取者の反応形式

- 発話終了後、聴取者7名全員が順に「関心を持った一点」を15〜20秒で述べる
- 内容：「私が一番気になったのは〜の部分です」「〜という表現が印象に残りました」
- 評価・感想・理由説明は不要、指示のみ
- 順序は運営が指定（ランダム）

### 6.6 欠席時の輪番調整

- 基本：欠席者を次回以降に後送り、順序を詰める
- 例外：1巡目冒頭のE2a・E2bの欠席は運営が手動判断（反転効果の維持）
- 2巡目・3巡目の欠席は運営が手動で調整

### 6.7 カメラ・マイクの扱い

- カメラ：全員オン
- マイク：発話者・関心表明中の聴取者のみオン、他はミュート
- チャット：使用しない（口頭のみで完結）

### 6.8 運営の立ち回り

- 発話内容への評価コメントは一切しない
- 時間管理と進行の整流のみ
- グランドルール第3条に基づき、進行について質問されたら根拠を答える

## 7. 第二フェーズ詳細設計

### 7.1 基本情報

- 期間：7月〜9月（3か月）
- 頻度：週3回（火・木・金）×12週＝36回
- 1回時間：15分
- 形式：通信会議
- 発話者A：1名（輪番）
- 応答者B：1名（運営指名）
- 聴取者：6名
- 運営：ゆ氏
- 1人あたり発話回数：4〜5回

### 7.2 1回の進行（15分）

| 区分 | 時間 | 内容 |
|------|------|------|
| 冒頭 | 30秒 | 運営が発話者A・応答者B・主題を告知 |
| 発話A | 6分 | 発話者Aが主題について話す |
| 問いを置くB | 2分 | 応答者Bが自分の中に生まれた問いを場に置く |
| Aの応答（任意） | 2〜3分 | Aが応えたい場合のみ応える |
| 聴取者6名の感想（任意） | 3〜4分 | チャットまたは口頭で、任意で感想を残す |
| 締め | 30秒〜1分 | 運営が締めの挨拶、次回の告知 |

### 7.3 ペアリング設計

**原則**

1. 等級差は2以内を基本とする
2. E5がE2の応答者になる組み合わせは避ける（E2が発話者のときの応答者にE5を置かない）
3. 下位者が発話者のとき、応答者に上位者を置くことで「受け取られる経験」を作る
4. ペアの固定化を避ける（3か月で全員が最低3〜4人と異なる応答者を経験）

**応答者候補の範囲**

| 発話者 | 応答者候補 |
|--------|-----------|
| E5 | E4p, E4, E3a, E3b, E3c |
| E4p | E5, E4, E3a, E3b, E3c, E2a, E2b |
| E4 | E5, E4p, E3a, E3b, E3c, E2a, E2b |
| E3（a/b/c） | E5, E4p, E4, 他のE3, E2a, E2b |
| E2（a/b） | E4p, E4, E3a, E3b, E3c, 他のE2（E5は置かない） |

### 7.4 主題（12週分）

| 週 | 主題 |
|----|------|
| 第1週 | 最近の業務で「もう少しこうだったら」と思ったこと |
| 第2週 | 新人だったとして、今の事務所をどう感じるか |
| 第3週 | 自分が担当していない業務領域で関心があるもの |
| 第4週 | 最近、自分の判断が鈍ったと感じた場面 |
| 第5週 | 自分が大事にしたい仕事の進め方 |
| 第6週 | 今の業務で「これは手放してもいい」と思う作業 |
| 第7週 | 自分が苦手意識を持っている業務の種類 |
| 第8週 | 最近、他の人のやり方で「いいな」と思ったもの |
| 第9週 | 自分が仕事で時間を使いたい領域 |
| 第10週 | 最近、自分が変わったと感じること |
| 第11週 | 今の事務所で「あってほしい」と思うもの |
| 第12週 | 第二フェーズ全体を振り返って、気づいたこと |

主題の選定基準：業務文脈の語彙が使える。正解がなく、意見の是非が問えない。自分の判断基準や考え方が自然に現れる。

### 7.5 「問いを置く」形式

**形式の定義**

発話者Aの発話を受けて、応答者Bが「自分の中に生まれた問い」を場に置く。Aへの質問ではなく、B自身の問いの表明。

**形式のポイント**

- Aの発話への同意でも反論でもない
- B自身の内省の表明
- Aに答えを求めない
- 場に開かれている（他の聴取者も考えられる問いになっている）

**例**

Aの発話：「最近の業務でもう少しクライアントとの距離感を変えたいと思った」

Bの問い：「私も距離感の調整で迷うことがある。近づきすぎても遠すぎても難しい、という問いがある」

### 7.6 発話者Aの応答権限

Aが応えるか否かは完全にA自身の裁量。

- 応えない場合：Bの問いが終わって数秒待ち、運営が「ありがとうございました」で次のステップへ
- 応える場合：2〜3分の時間内で、Bの問いに対する自分の考え、または自分の中に新たに生まれた問いを返す

### 7.7 聴取者6名の役割

- 発話Aと問いBを聴いた後、感じたことがあれば口頭またはチャットで残す
- 沈黙のまま終わることも許容
- 運営からの指名はしない

### 7.8 フェーズ移行時の告知

第二フェーズ開始時（7月第1週火曜）の冒頭で、運営から明示的に告知。

- 第一フェーズの2か月が終わったこと
- 第二フェーズの目的（業務に近接する領域で、問いを交わす経験を作る）
- 形式の変更点
- 期間は3か月、9月末まで

この回のみ15分+告知5分で20分程度となる。例外として許容。

### 8.3 移行時に必要な作業

- 議論のルールを改めて場に共有
- 第二フェーズまでに醸成された関係性の質を毀損しないための作法を設計

## 9. 運営体制

### 9.1 運営担当

- ゆ氏（等級5とは別人格、独立した運営担当として機能）

### 9.2 運営の位置取り

- 等級5の発起意図を理解しつつ、中立的な進行役として機能する
- 職制的権威ではなく、グランドルールへの準拠によって運営権限の正当性を担保する
- 判断の予測可能性を担保することで、他メンバーの信頼を得る

### 9.3 運営の主な作業

**日次**

- 当日の朝礼の進行
- 出欠の確認
- 欠席通知の受信

**週次**

- 翌週の主題の確定・通知
- 翌週の発話者（2巡目以降）・応答者（第二フェーズ）の決定・通知

**月次**

- 輪番計画の見直し
- 特性メモの更新（アプリ外で個別管理）

**フェーズ切替時**

- 第一フェーズ終了時（6月末）：運営内棚卸し、第二フェーズのペアリング設計
- 第二フェーズ終了時（9月末）：第三フェーズへの移行設計

### 9.4 運営のメモの扱い

- 参加者の特性メモはアプリ外（メモアプリ・紙等）で運営が個別に管理
- 運営の個人判断のために使用
- 体系的なデータベース化は行わない

## 10. 試走計画

### 10.1 試走の目的

本番開始前に、10分構成の時間配分が現実的かを運営側で確認する。

### 10.2 実施方法

- 形式：運営単独シミュレーション
- 回数：本番前1回
- タイミング：4月末週
- 場所：本番と同じ通信会議ツールを使用
- 所要時間：20〜30分程度

### 10.3 検証項目

| 項目 | 検証内容 |
|------|---------|
| 冒頭 | 30秒で告知が成立するか |
| 発話 | 5分の時間感覚が妥当か |
| 関心表明 | 2分で7名が回るか |
| 締め | 定型化できるか |
| ツール操作 | 運営が進行しながら操作できる負荷か |

### 10.4 試走後の設計変更権限

運営単独に付与。変更した場合は以下を記録する。

- 変更前の設計
- 変更後の設計
- 変更理由（試走で観察された事象）

### 10.5 変更時の優先順位

機能しない場合の対処順序。

1. 最優先：発話5分の確保（短縮しない）
2. 第二優先：関心表明の全員参加
3. 第三優先：冒頭・締めの簡潔さ
4. 第四優先：総時間の10分厳守（12分程度までは許容）

## 11. 出欠管理アプリ

### 11.1 アプリの位置づけ

朝礼の運営効率化のための支援ツール。別途antigravityで開発する。

### 11.2 MVPの範囲（5月開始前）

- 認証機能（運営・参加者の権限分離）

- 本人からの申し出がない限り、それ以前は接触しない

## 12. リスクと留意点

### 12.1 設計上の残リスク

**リスク一　等級5の振る舞いへの依存**

等級5の1名が発起人かつフランクな姿勢との前提で設計しているが、この姿勢が揺らぐと全設計が機能しない可能性がある。継続的な観察が必要。

**リスク二　「聴取者の15秒関心表明」の形骸化**

3巡目あたりから「表明のための表明」に陥る可能性。特に後半の主題で関心の表明が深くなる必要があるが、形式が固定化すると深まりが阻害される。

**リスク三　業務文脈への情報漏洩**

前文から「業務評価と独立」を削除したため、業務の場で朝礼内容が参照されるリスクは残る。等級4+・E4への個別の趣旨共有が必要。

**リスク四　9月末到達の時期優先リスク**

第一・第二フェーズの状態基準を満たさないまま9月末を機械的に迎えると、議論が表層化し、関係性の質を損なう逆効果になり得る。時期と状態基準の両方を満たすまで延長する判断も必要。

**リスク五　第二フェーズの「問いを置く」形式の機能**

実際の運用で機能するかは、第一フェーズで醸成される関係性の質に依存する。第一フェーズ終了時に初めて検証可能。

### 12.2 運用上の留意点

- 主題は週単位で運営が指定、火曜を週の起点とする
- 祝日は朝礼非実施、発話予定は翌実施日に繰り延べ
- 欠席理由は問わない運用を徹底する
- 運営は第3条に基づき、判断根拠を常に説明できる状態を維持する

## 13. 重要な前提と検証事項

### 13.1 事実と推定の区別

**事実（確認済）**

- 仙台事務所メンバー8名の等級構成
- 等級5が朝礼の発起人である
- 参加者は全員、対話を業とする財務コンサル
- グランドルールの最終案
- 3フェーズの期間配分

**推定（未検証）**

- 「関係性構築の優先度が低い」タイプのメンバーはいないという前提
- 現状「闊達な意見が出ない」主因は心理的安全性・議論の作法不慣れ・階層影響であるという仮説
- 等級5がグランドルールに間違いなく従うという見込み
- 第二フェーズの「問いを置く」形式が機能するという仮説

**不明（要確認）**

- 等級3・等級2の5名の朝礼という別文脈での振る舞い特性
- アプリの具体仕様（これからantigravityで開発）
- 試走での実際の時間配分の妥当性

### 13.2 最も不確実な前提

第一フェーズで醸成される関係性の質が、第二フェーズ以降の形式を支えられるかどうか。これは6月末の時点で初めて検証可能になる。

### 13.3 検証に必要な情報

- 第一フェーズ1巡目終了時の運営内棚卸し結果
- 第一フェーズ終了時の各メンバーの発話傾向の観察結果
- 試走での時間配分の妥当性

### 13.4 次に取るべき一手

1. antigravity向けプロンプトを使ってアプリ開発を開始
2. 4月末週に運営単独で試走実施
